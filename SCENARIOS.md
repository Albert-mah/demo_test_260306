# AI CRM Scenarios — 真实场景定义

## 运行模式

- **后台运行**：数据变更触发 → AI 处理 → 结果写入关联表 → 用户打开页面时看到
- **前台运行**：用户主动触发 → AI 实时处理 → 结果填入当前表单

## 通用模式

### 模式 A：后台预处理 + 旁显
场景：工单预处理(#2)、工单翻译(#3)、邮件翻译(#9)
流程：新记录 → 后台AI处理 → 结果存关联表 → 详情页旁边显示 → 一键插入/采纳

### 模式 B：AI建议表 + 人操作 + AI执行
场景：知识积累(#1)、满意度预警(#5)、背景调查(#7)、违规预警(#8)
流程：AI分析 → 写入建议/预警表(带状态) → 页面静默显示 → 人点操作 → AI执行后续

### 模式 C：语音/文字 → 正式双语输出
场景：工单回复助手(#4)、邮件回复助手(#10)
流程：用户输入中文大意(语音) → AI生成正式外语回复 + 中文审查版 → 填入表单

### 模式 D：文件分析 → 自动动作
场景：凭证分析(#6)
流程：上传文件 → AI解析 → 与业务数据对比 → 自动执行状态变更 + 触发子流程

### 模式 E：问答/查询
场景：邮件提问(#12)
流程：用户问问题 → AI 基于数据回答

## 场景详情

### 工单域

#### #1 工单知识积累（后台·模式B）
- 触发：工单标记为"已解决"
- AI做什么：分析工单内容 → 与知识库/文档比对 → 判断是否值得行动
- 产出：写入"知识建议表"，每条给3个操作选项：
  1. 积累到知识库
  2. 补充到使用手册
  3. 否决
- 人操作：点选其中一个 → AI 自动完成（写入知识库/手册对应位置）

#### #2 工单预处理（后台·模式A）
- 触发：新工单创建 / 工单有新回复
- AI做什么：根据知识库内容预生成回复（客户语言 + 中文版）
- 显示：查看工单详情时，回复框旁边显示预回复
- 操作：一键插入 / 不采纳
- 不采纳时：AI 根据实际回复学习/积累知识
- 为什么后台：前台太慢（点击到生成几十秒），预处理实现"打开就有"

#### #3 工单翻译（后台·模式A）
- 触发：新工单/新回复
- AI做什么：翻译为中文
- 显示：原内容旁边或底部
- 状态：已基本可用

#### #4 工单回复助手（前台·模式C）
- 触发：用户主动（语音输入中文大意）
- AI做什么：用客户语言写正式回复 + 中文审查版
- 产出：自动填入回复表单

#### #5 客户满意度预警（后台·模式B）
- 触发：定期/工单关闭时
- AI做什么：分析客户的工单+邮件历史 → 评估满意度
- 产出：风险客户写入预警表 → 推送

### 订单域

#### #6 凭证分析（后台·模式D）
- 触发：客户上传银行转账凭证
- AI做什么：OCR解析凭证 → 与订单数据对比 → 确认匹配
- 产出：自动标记已支付 → 自动开通权益（触发子流程）

### 客户域

#### #7 背景调查（后台·模式B）
- 触发：新客户创建
- AI做什么：中国客户用千问、海外用Gemini调查背景
- 产出：调查信息存入客户记录 + 高价值客户推送到预警/关注表

#### #8 违规使用预警（后台·模式B）
- 触发：定期/应用信息变更
- AI做什么：分析客户应用 + 许可证版本 + 背景 → 检测违规
- 产出：违规预警写入预警表

### 邮件域

#### #9 邮件翻译（后台·模式A）
- 触发：新邮件
- AI做什么：翻译为中文
- 显示：与原邮件并列（通过邮件翻译表关联邮件表）
- 重点：UI自定义实现

#### #10 邮件回复助手（前台·模式C）
- 触发：用户主动（语音/移动端）
- AI做什么：中文大意 → 客户语言正式回复 + 中文审查版
- 产出：填入表单

#### #11 邮件总结（后台·模式B变体）
- 触发：客户邮件数 ≥ 5 / 新邮件到达
- AI做什么：总结该客户所有邮件往来
- 产出：存到客户下，增量追加

#### #12 邮件提问查询（前台·模式E）
- 触发：用户主动
- AI做什么：基于全部邮件或单客户邮件回答问题
