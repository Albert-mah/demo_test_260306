# AI CRM Demo — 交互设计规范

从工单模块的实际实现中提炼，作为所有页面的设计基准。

---

## 核心概念：AI 员工

每个 AI 能力不是一个"功能按钮"，而是一个**有身份的员工**。

| 员工 | emoji | 色值 | 能力 | 出现场景 |
|------|-------|------|------|---------|
| 翻译专员 | 🌐 | #1677ff | translate | 工单语言检测/翻译 |
| 分类专员 | 🏷️ | #722ed1 | classify | 工单分类 |
| 优先级顾问 | 🚦 | #fa541c | decide | 工单优先级 |
| 回复助手 | ✍️ | #52c41a | generate | 工单回复草稿 |
| 知识管家 | 📚 | #faad14 | decide | 已解决工单→知识积累 |
| 情报分析师 | 🔍 | #13c2c2 | investigate | 客户背景调查 |
| 邮件秘书 | 📧 | #eb2f96 | summarize | 邮件摘要/问答 |
| 财务核对员 | 💰 | #faad14 | validate | 凭证核对/风控 |
| 客户成功顾问 | 😊 | #52c41a | decide | 满意度/流失风险 |
| 合规审查员 | 🛡️ | #f5222d | validate | 许可证合规 |

**团队模式**：多个员工协作时，用 `AITeamAvatars` / `AIFusedAvatar` 展示。
- collaborative（融合头像）= 一次 AI 调用，合并 prompt
- parallel（并排头像）= 各自独立调用

## 视觉系统

### 紫色 = AI

统一色值 `#8b5cf6`，所有 AI 产出的内容用紫色系标识：

```
背景色：#faf8ff（极浅紫）
左边框：3px solid #8b5cf6（实紫）或员工专属色
渐变底：linear-gradient(135deg, #faf8ff, #f5f0ff)
边框色：#e8e0f7 / #f0ecff / #d8b4fe
```

### AI 填充字段样式

当某个字段值是 AI 生成的，用以下样式标记：

```css
.ai-field {
  background: #faf8ff;
  border-left: 3px solid #8b5cf6;  /* 或员工专属色 */
  padding: 2px 8px;
  border-radius: 0 4px 4px 0;
}
```

### Hover 渐显

AI 元素不抢注意力——默认隐藏，hover 时渐显：

```css
.ai-cell-wrapper .ai-cell-icon { opacity: 0; transition: opacity 0.2s; }
.ai-cell-wrapper:hover .ai-cell-icon { opacity: 1; }

/* 表格行级：hover 行时半透明显示，hover 具体单元格时全显 */
.ant-table-row:hover .ai-cell-icon { opacity: 0.6; }
.ant-table-row:hover .ai-cell-wrapper:hover .ai-cell-icon { opacity: 1; }
```

## 交互分层

从小到大，5 个层级的 AI 交互入口：

### 1. 字段级（AICell / AIField）

**位置**：表格单元格、详情页 Descriptions 内

**模式**：
- 默认只显示字段值 + 紫色底色标记
- 鼠标移入行 → AI 头像半透明浮现
- 鼠标移入具体字段 → AI 头像完全显示
- 点击头像 → `AIResultPopover` 弹出

**组件**：
- `AICell`（TicketList 用）：表格内，带 hover 逻辑
- `AIField`（TicketDetail 用）：详情页内，同样 hover 逻辑
- `AIFieldBadge`：纯视觉标记，不含交互

**代码模式**：
```tsx
<span className="ai-cell-wrapper" style={{ background: '#faf8ff', borderLeft: '2px solid #d8b4fe', ... }}>
  {value}
  <AIResultPopover results={results} tasks={tasks} onRefresh={load}>
    <span className="ai-cell-icon" style={{ opacity: 0, transition: 'opacity 0.2s' }}>
      <AIAvatar avatar={task.avatar} color={task.avatar_color} size={16} />
    </span>
  </AIResultPopover>
</span>
```

### 2. 卡片级（AI 员工卡片）

**位置**：详情页主要内容区，与业务数据并排

**模式**：
- 有明确的 AI 员工 header（头像 + 名称 + 描述）
- 内容区展示 AI 产出（多选项/分析/建议）
- 底部有对话输入框

**实例**：
- `AIReplyCard`（TicketDetail 右侧）：回复助手卡片，展示语气选项
- AI 360 面板（CustomerDetail 右侧）：多员工协作分析
- 邮件秘书 Q&A（CustomerDetail 右侧）：对话式交互

**卡片结构**：
```
┌─ 渐变 header ─────────────────────┐
│ [AI头像] 员工名称                    │
│          描述文字                    │
├────────────────────────────────────┤
│ 内容区（选项卡片 / 分析文本 / 图表） │
│                                    │
│ [对话输入框] [发送]                  │
└────────────────────────────────────┘
```

### 3. 悬浮洞察（Hover Popover）

**位置**：表格中的客户名、满意度分数、风控标记等

**模式**：
- 不需要点击，hover 0.2~0.3s 后自动弹出
- 轻量信息卡片（240~280px 宽）
- 包含 AI 员工头像 + 1~2 行分析

**实例**：
- `CustomerHoverCard`：客户名 hover → mini 360（工单/订单/满意度 + 情报分析师背景）
- 满意度 hover → 客户成功顾问风险分析
- 订单 hover → 财务核对员风控评估
- AI 处理进度 hover → 各步骤 ✓/⏳ 状态

### 4. 选中文字触发（AISelectionBar）

**位置**：详情页内容区，用户拖选文字后浮现

**模式**：
- 鼠标选中文字 → 选区上方出现浮动操作条
- 快捷操作：翻译 / 摘要 / 改写 / 解释 / 对话
- 右侧显示相关 AI 员工头像，可指定员工

### 5. 全局浮动按钮（AIFloatingButton）

**位置**：页面右上角固定位置

**模式**：
- 显示当前页面参与的 AI 团队头像（融合模式）
- 右上角 Badge 显示待处理数量
- 点击展开面板：
  - 顶部：AI 员工筛选器（点击头像 toggle 过滤）
  - Tab 1 "待处理"：待采纳/待拒绝的 AI 结果列表
  - Tab 2 "全部"：时间线 / 列表视图切换
  - 对话模式：选中某员工开始对话

## AI 结果生命周期

```
触发 → AI 执行 → 结果写入(pending) → 人工决策 → 终态
                                        ├── 采纳(applied)
                                        ├── 拒绝(rejected)
                                        ├── 编辑后采纳(modified)
                                        └── 重跑(retry → new pending)
```

### AIResultPopover 统一交互

所有 AI 结果的操作入口，无论从哪个层级触发，弹出后的交互一致：

```
┌─ [AI头像] 员工名 → field_name ─── [状态Tag] ─┐
│                                                │
│ ┌─ 紫色底色 ──────────────────────────────┐   │
│ │ AI 生成的内容                            │   │
│ └──────────────────────────────────────────┘   │
│                                                │
│ gemini-2.0-flash · 95% · 186t · 1240ms        │
│                                                │
│ [采纳] [拒绝] [编辑] [重跑] [复制]             │
├────────────────────────────────────────────────┤
│ ▸ 查看完整上下文                               │
│ ▸ 展开对话微调 (含上下文)                       │
└────────────────────────────────────────────────┘
```

## 工作流编辑器（WorkflowEditor）

三栏布局：工作流列表（左）→ 可视化节点链（中）→ 执行记录（右）。

### 布局结构

```
┌─ 工作流列表 ─┐  ┌─ 可视化节点链 ────────────────┐  ┌─ 执行记录 ──────┐
│ ▸ 工单全流程   │  │ [触发器: 记录创建]              │  │ ● completed     │
│   工单分类     │  │       ↓                        │  │   03-04 14:22   │
│   工单翻译     │  │ [AI任务: 翻译+分类]             │  │ ● completed     │
│               │  │       ↓                        │  │   03-04 14:18   │
│ 选中项紫色     │  │ [条件: priority == urgent]      │  │ ● failed        │
│ 左边框 #8b5cf6 │  │       ↓                        │  │   03-04 13:55   │
│               │  │ [通知: 发送邮件]                │  │                 │
│               │  │       ↓                        │  │ (点击查看详情)   │
│               │  │ [结束]                          │  │                 │
└───────────────┘  └─────────────────────────────────┘  └─────────────────┘
```

### 节点类型

| 类型 | 图标 | 色值 | 用途 |
|------|------|------|------|
| trigger | ⚡ | #f5222d | 触发器（记录创建/更新/定时等） |
| ai_task | 🤖 | #8b5cf6 | AI 任务节点，可选多个员工 |
| condition | ⑂ | #faad14 | 条件判断（字段 + 运算符 + 值） |
| action | ▶ | #1677ff | 数据操作（更新/创建记录） |
| notification | 🔔 | #52c41a | 通知（邮件/消息） |
| delay | ⏱ | #999 | 延时等待 |
| end | ✓ | #999 | 结束节点 |

### AI 任务节点 — 团队模式

当一个 AI 任务节点选择了多个员工，自动出现模式选择器：

```
┌─ AI 任务: 翻译+分类 ────── [AI 任务] ─┐
│ ┌─ 紫色底色 (#faf8ff) ───────────────┐ │
│ │ [融合头像] 协作                      │ │
│ │ 合并 Prompt → 1 次调用              │ │
│ │ ─────                              │ │
│ │ 🌐 翻译专员  将文本翻译为...         │ │
│ │ 🏷️ 分类专员  对工单进行分类...       │ │
│ │ ─────                              │ │
│ │ 合并输出: {"language","translated"}  │ │
│ └────────────────────────────────────┘ │
└────────────────────────────────────────┘
```

**模式选择器**（节点编辑 Drawer 内）：

```
┌──── 协作 ─────────────────┐  ┌──── 并发 ─────────────────┐
│ [融合头像]  协作             │  │ [并排头像]  并发             │
│ 合并 Prompt → 1 次调用      │  │ N 次独立调用 → 各自输出      │
│ → JSON 合并输出             │  │ → 按字段回填                │
│                            │  │                            │
│ ██ 选中时紫色边框           │  │    未选中灰色边框            │
└────────────────────────────┘  └────────────────────────────┘
```

- **collaborative（协作）**：合并所有员工的 system prompt + user prompt → 一次 AI 调用 → JSON 合并输出
- **parallel（并发）**：每个员工独立调用 → 各自输出 → 按字段回填

## 联动规则配置（RuleConfigPanel）

页面级的 AI 自动化入口，配置"什么时候 → 哪些 AI 员工 → 结果怎么处理"。

### 规则结构

每条规则是一个三段式流程：

```
触发条件  →  AI 员工（可多选组团）→  结果处理
```

### 触发条件分组

| 分组 | 触发项 |
|------|--------|
| 数据事件 | 记录创建时 / 记录更新时 / 字段变更时 |
| 用户交互 | 字段聚焦时 / 选中文字时 / 右键菜单 / 按钮点击时 / 鼠标悬停时 |
| 页面/区块 | 页面打开时 / 区块渲染时 / 列表加载时 |
| 定时/工作流 | 定时任务 / 工作流节点 |

### 结果处理

| 动作 | 说明 |
|------|------|
| 写入字段 | 直接写入指定字段（需指定目标字段名） |
| 气泡提示 | 显示 AI 结果，等待人工采纳/拒绝 |
| 触发工作流 | 将结果传给后续工作流 |
| 发送通知 | 邮件/消息通知 |
| 创建预警 | 写入预警中心 |

### 规则卡片样式

```
┌─ #1 [开关] [AI团队头像] AI 团队 ─── [删除] ─┐
│                                               │
│ ⚡ 触发条件       → 🤖 AI 员工        → 结果   │
│ [记录创建时 ▼]     [翻译专员 ✕]        [气泡 ▼] │
│                    [分类专员 ✕]                 │
│                    [优先级顾问 ✕]               │
│                    多个员工自动合并为            │
│                    单次 AI 调用                 │
└────────────────────────────────────────────────┘
```

- 规则启用时：紫色左边框 `3px solid #8b5cf6`
- 规则禁用时：灰色左边框 + `opacity: 0.6`
- 多选员工自动显示 `AITeamAvatars` 和 "AI 团队" 标签

## 设计原则

1. **AI 不抢注意力** — 默认隐藏细节，hover 半透明，hover 目标全显，点击展开
2. **紫色 = AI** — 一眼区分人填的和 AI 填的，不需要读文字
3. **人始终有控制权** — 每个 AI 结果都可以采纳/拒绝/编辑/重跑/对话微调
4. **员工有身份** — 不是匿名的"AI 建议"，是具体的"🌐翻译专员"，可追溯
5. **渐进式展开** — 字段标记 → hover 头像 → 点击 popover → 展开对话
6. **上下文自动携带** — 所有 AI 操作默认携带当前页面/记录上下文，无需手动粘贴
